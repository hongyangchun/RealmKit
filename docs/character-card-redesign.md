# 人物卡片重设计方案：三国杀风格深度分析与提案

> **分析日期**：2026-06-13 | **分析范围**：三国杀武将牌标准设计 vs RealmKit CharacterCard 现状  
> **目标用户**：12岁少年创作者 | **设计原则**：游戏感、沉浸式、可打印

---

## 一、三国杀武将牌完整解剖

### 1.1 卡牌物理规格

| 属性 | 三国杀标准 | RealmKit 现状 | 差距 |
|------|-----------|-------------|------|
| 比例 | **5:7** (63×88mm) | 最大宽度300px，高度可变 | ❌ 非标准比 |
| 材质感 | 亚麻纹理卡纸 | CSS repeating-linear-gradient 斜纹 | ✅ 已有纹理 |
| 边框 | 势力色外框 + 金色内框 + 装饰角纹 | 势力色 2px 边框 + 四角L纹 | ⚠️ 层次不足 |
| 圆角 | 8-12px 大圆角 | 8px | ✅ 匹配 |

### 1.2 卡面区域完整解剖（12区模型）

三国杀标准武将牌从上到下可分解为 **12 个设计区域**：

```
┌──────────────────────────────────┐
│ Z1  势力徽记 + 势力名 (左上角)     │  ← 小字，势力色
│ Z2  双势力色线 (顶部装饰条)        │  ← 标志性设计元素
│ Z3  称号/别名 (右上区域)           │  ← e.g.「乱世奸雄」「武圣」
├──────────────────────────────────┤
│                                  │
│ Z4  武将名 (超大字体，居中)        │  ← 视觉重心 #1
│     ┌────────────────────┐       │
│     │                    │       │
│ Z5  │   立绘/角色插画     │       │  ← 占卡面 55-65%
│     │   (全身或半身像)    │       │    视觉重心 #2
│     │                    │       │
│     └────────────────────┘       │
│                                  │
│ Z6  体力值 · 勾玉/阴阳鱼          │  ← 右侧竖向排列 3-5 个
├──────────────────────────────────┤
│ Z7  技能标题 (势力色标签)         │
│ Z8  技能描述文字 (完整段落)        │  ← 占卡面 20-25%
│ Z9  技能标题 + 描述 (第2技能)     │
│ Z10 技能标题 + 描述 (第3技能)     │
├──────────────────────────────────┤
│ Z11 稀有度/编号/画师              │
│ Z12 势力色底栏 + 卡牌编号         │  ← 底部收束
└──────────────────────────────────┘
```

### 1.3 三国杀势力配色体系

| 势力 | 主色 | 辅色 | 象征 |
|------|------|------|------|
| **魏** (Wei) | `#1a4d8c` 蓝 | `#2b6cb0` | 海、权谋 |
| **蜀** (Shu) | `#cc2936` 赤 | `#e53e3e` | 火、忠义 |
| **吴** (Wu) | `#2d8c4e` 绿 | `#38a169` | 林、水军 |
| **群** (Qun) | `#6b5a8a` 紫灰 | `#8b7aa8` | 大地、诸侯 |
| **神** (God) | `#d4af37` 金 | `#f6e05e` | 天、传说 |

### 1.4 勾玉（体力值）设计规范

三国杀最具标志性的视觉元素：

- **形状**：逗号形/阴阳鱼形（magatama），由一个大圆和一个小圆通过曲线连接
- **颜色**：填充为金色/黄色渐变，轮廓为势力色
- **排列**：竖向排列在卡牌右侧，间距紧凑
- **数量**：3勾玉（文官）= 3体力，4勾玉 = 4体力，5勾玉（猛将）= 5体力
- **语义**：代表角色的生命值/耐久度/战斗力

### 1.5 技能展示规范

三国杀技能的呈现方式非常讲究：

- **技能名**：用势力色的背景标签展示，字体加粗
- **技能描述**：完整的规则文字，包含触发条件、效果、限制
- **格式模板**：`[时机]，你可以[条件]→[效果]。每回合限X次。`
- **分隔**：技能之间用势力色细线分隔
- **字体**：宋体/楷体，字号比描述文字大一号

## 二、当前 RealmKit CharacterCard 差距分析

### 2.1 区域对比矩阵

| 三国杀区域 | 当前实现 | 匹配度 | 优先级 | 说明 |
|------------|---------|--------|--------|------|
| Z1 势力名 | 标题栏内势力徽章 | 🟡 50% | P1 | 位置不对，不够显眼 |
| Z2 双势力线 | 标题栏渐变背景 | 🟡 40% | P1 | 渐变替代了双线，视觉效果不同 |
| Z3 称号 | 与名字同行 `.title` | 🟡 60% | P2 | 太小，不够突出 |
| Z4 武将名 | 标题栏内 1.15rem 白字 | 🔴 30% | **P0** | **太小！应占卡面主体** |
| Z5 插画 | 80px 高方形 Avatar | 🔴 15% | **P0** | **严重不足！应占55-65%** |
| Z6 勾玉体力 | **不存在** | 🔴 0% | **P0** | **核心缺失** |
| Z7-10 技能文字 | Compact SkillChip 标签 | 🔴 25% | **P0** | **应展示完整描述** |
| Z11-12 底栏 | **不存在** | 🔴 0% | P2 | 缺少编号/稀有度 |

### 2.2 数据模型差距

当前 `Character` 类型缺少三国杀核心属性：

```typescript
// 当前 Character 接口
interface Character {
  id: string;
  name: string;
  factionId: string;
  avatar?: string;        // ← 只有小头图，无全身立绘
  birthYear?: number;
  deathYear?: number;
  title?: string;         // ← 职衔（如"大将军"），非称号（如"乱世奸雄"）
  skills: Skill[];        // ← 有技能，但卡片上用 Chip 展示
  traits: string[];
  bio: string;
  relatedEventIds: string[];
}

// 三国杀核心属性缺失：
// ❌ hp / maxHp       — 体力值（1-8）
// ❌ nickname         — 称号（区别于 title/职衔）
// ❌ portrait         — 全身立绘（区别于 avatar/头像）
// ❌ cardNumber       — 卡牌编号
// ❌ rarity           — 稀有度
```

### 2.3 视觉层级对比

| 视觉元素 | 三国杀权重 | 当前权重 | 差距 |
|----------|-----------|---------|------|
| 名字 | 极重（30-36px，视觉重心） | 轻（1.15rem，约18px） | **-50%** |
| 插画 | 极重（占55-65%） | 极轻（80px = ~20%） | **-65%** |
| 技能 | 重（完整文字描述） | 轻（紧凑 Chip） | **信息量-80%** |
| 体力 | 重（勾玉图标） | 无 | **完全缺失** |
| 称号 | 中 | 轻 | - |
| 边框 | 重（3层） | 中（2层+L纹） | - |

## 三、新方案提案：Enhanced 三国杀 CharacterCard v2.0

### 3.1 数据模型扩展

```typescript
// 新增类型
export type CardRarity = 'common' | 'rare' | 'epic' | 'legendary';

// 扩展后的 Character 接口
export interface Character {
  // ===== 现有字段 =====
  id: string;
  name: string;
  factionId: string;
  avatar?: string;
  birthYear?: number;
  deathYear?: number;
  title?: string;
  skills: Skill[];
  traits: string[];
  bio: string;
  relatedEventIds: string[];

  // ===== 新增字段 =====
  /** 体力值 (1-8, 默认4) — 对应三国杀勾玉数量 */
  hp: number;
  /** 称号/别名 — e.g. "乱世枭雄"、"武圣"，区别于职衔 */
  nickname?: string;
  /** 全身/半身立绘 (base64) — 区别于头像小图 */
  portrait?: string;
  /** 卡牌编号 — e.g. "WEI-001" */
  cardNumber?: string;
  /** 稀有度 */
  rarity?: CardRarity;
}
```

### 3.4 勾玉（体力值）设计方案

勾玉使用纯 CSS/SVG 实现：

**方案A — SVG Path**（推荐，导出友好）：
```svg
<!-- 单枚勾玉 SVG -->
<svg viewBox="0 0 24 24" width="20" height="20">
  <!-- 大圆 -->
  <circle cx="12" cy="10" r="8" fill="url(#magatama-gold)" stroke="currentColor" stroke-width="1"/>
  <!-- 小圆 -->
  <circle cx="12" cy="17" r="3.5" fill="url(#magatama-gold)" stroke="currentColor" stroke-width="1"/>
  <!-- 连接曲线 -->
  <path d="M6.5 13 Q9 16 12 17" fill="none" stroke="currentColor" stroke-width="1"/>
  <path d="M17.5 13 Q15 16 12 17" fill="none" stroke="currentColor" stroke-width="1"/>
</svg>
```

**方案B — CSS 纯绘**（React友好）：
使用 Box 组件的 border-radius + 绝对定位叠加实现阴阳鱼形状。

**颜色规范**：
- 满体力：金色渐变填充 `#e8c547` → `#c9a050`，边框 `#a08030`
- 空体力：灰色 `#555`，边框 `#333`
- 当前体力位置：用半透明势力色覆盖表示

## 五、设计决策记录

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 卡牌比例 | 固定5:7 / 自适应 | **固定5:7** | 三国杀标准，适合打印 |
| 勾玉实现 | SVG / CSS纯绘 / Emoji | **SVG Path** | 导出友好，精确控制 |
| 立绘 vs 头像 | 合并 / 分离 | **分离** | 卡片用立绘(全身)，列表用头像(缩略图) |
| 称号 vs 职衔 | 合并 / 分离 | **分离** | 三国杀称号≠职衔，语义不同 |
| 稀有度 | 手动 / 自动计算 | **自动为主+手动覆盖** | 减少用户负担，保留定制空间 |
| 深色/浅色 | 仅深色 / 双主题 | **深色默认 + 可切换** | 游戏感优先，但保留可读性 |
