# PRD：世界圣典（ZZWorld Chronicle）

**版本**：v1.0  
**产品经理**：许清楚  
**日期**：2025-07-17  
**语言**：中文  
**技术栈**：Vite + React + MUI + Tailwind CSS  
**项目名称**：zzworld_chronicle  

---

## 一、原始需求

用户（父亲/开发者）希望为其12岁儿子开发一款专为历史架空世界创作设计的 Web 工具——「世界圣典」。该孩子已自发设计了一张手绘架空世界地图，3-6 个国家，数十个人物（风格类似三国杀人物卡片），若干历史事件与时间线，以及手绘人物关系图。当前痛点是记录零散、容易遗忘、时间线与人物设计存在冲突。

---

## 二、产品目标

1. **集中管理世界创作内容**：提供统一的数字化工作台，将地图、人物、事件、关系等创作内容结构化存储，替代零散纸质记录。
2. **提供游戏风格的沉浸体验**：以三国杀卡片风格呈现人物，让12岁用户感受"自己世界的史书"般的成就感与创作乐趣。
3. **保障数据安全与可迁移性**：本地优先存储（localStorage），支持 JSON 全量导入导出，父亲可随时备份或迁移数据。

---

## 三、用户故事

| # | 角色 | 需求 | 价值 |
|---|------|------|------|
| US-1 | 12岁创作者 | 上传我的手绘地图，并在地图上打图钉标记国家和城市 | 把纸上的世界数字化，随时查看和补充 |
| US-2 | 12岁创作者 | 创建三国杀风格的人物卡片，填写技能标签和人物传记 | 让我的人物有"正式感"，像真正的游戏卡牌一样 |
| US-3 | 12岁创作者 | 在时间轴上记录历史事件，当我把同一人物放在矛盾时间时得到警告 | 避免剧情冲突，让世界观更严密 |
| US-4 | 12岁创作者 | 看到自动生成的人物关系网络图，可以按国家筛选 | 快速回顾复杂的人物关系，不用重新翻手绘图 |
| US-5 | 父亲/开发者 | 一键导出整个世界的 JSON 数据，也能导入备份 | 确保孩子的创作成果不丢失，方便迁移到未来的新版本 |

## 四、需求池

### P0（必须有 · MVP 核心）

| ID | 功能 | 描述 |
|----|------|------|
| P0-1 | 人物卡片 CRUD | 创建/编辑/删除人物卡片，包含：头像上传、姓名、国籍、技能标签（多选）、特质标签、人物传记文本 |
| P0-2 | 历史时间轴 | 按年份排列的可视化时间轴；支持新增/编辑/删除事件；每条事件可关联人物 |
| P0-3 | 冲突检测 | 同一人物在同一时间段出现于逻辑矛盾事件时，自动高亮警告（如：同年出现在两场不同地点的关键战役） |
| P0-4 | JSON 导入/导出 | 一键将所有数据（地图图钉、人物、事件、关系）导出为单一 JSON 文件；支持 JSON 导入恢复数据 |
| P0-5 | 本地持久化 | 所有数据优先存储在 localStorage，无需登录，离线可用 |

### P1（应该有 · 重要体验）

| ID | 功能 | 描述 |
|----|------|------|
| P1-1 | 世界地图模块 | 上传手绘地图图片作为背景；在地图上拖放图钉标记地点；点击图钉展示关联信息 |
| P1-2 | 多历史时期图层 | 地图支持切换不同时代的图层（如：建国前 / 鼎盛期 / 战乱期），图钉按时期显示/隐藏 |
| P1-3 | 关系网络图 | 根据人物卡片中的关系字段自动生成可视化关系图（力导向图）；支持按国家过滤；关系类型自定义（盟友/对手/师徒等） |
| P1-4 | 首页总览仪表盘 | 展示：世界名称、国家数、人物数、事件数、最近编辑记录，作为进入各模块的入口 |
| P1-5 | 全局搜索 | 顶栏搜索框，跨人物/事件/地点全文检索，结果直接跳转 |

### P2（可以有 · 加分项）

| ID | 功能 | 描述 |
|----|------|------|
| P2-1 | 人物卡片打印/分享 | 将单张人物卡片渲染为图片（PNG），可保存或分享 |
| P2-2 | 国家/势力管理 | 独立的国家/势力页面，含国旗色彩配置、简介、领土范围标记 |
| P2-3 | 时间轴多维过滤 | 时间轴支持按国家、人物、事件类型过滤显示 |
| P2-4 | Cloudflare D1 云端同步 | 可选的云端存储后端，支持多设备访问（父亲后续扩展） |
| P2-5 | 暗色模式 | 支持深色主题，更符合游戏风格审美 |

---

### 5.3 世界地图模块

- 主体为全屏地图画布，背景为上传的手绘图片
- 右上角：时期图层切换下拉选择器
- 工具栏：选择模式 / 添加图钉 / 删除图钉
- 图钉样式：圆形彩色 pin，悬停显示地名，点击弹出侧边抽屉展示关联人物和事件
- 左下角：图钉图例（国家颜色对应）

---

### 5.4 人物卡片库

- 顶部：筛选栏（按国家/势力过滤）+ 「新建人物」按钮 + 卡片/列表视图切换
- 主体：卡片网格布局（3列），每张卡片模拟三国杀风格：
  ```
  ┌──────────────────┐
  │   [头像区域]      │  ← 左侧竖向彩色国家色带
  │  姓名 · 国籍      │
  │  ──────────────  │
  │  [技能1] [技能2]  │  ← 技能标签 badge
  │  [特质标签]       │
  │  ──────────────  │
  │  简短传记摘要...  │
  └──────────────────┘
  ```
- 点击卡片：展开详情抽屉，含完整传记、关联事件列表、关系列表
- 编辑卡片：内联表单，支持头像上传（本地图片转 base64）

---

### 5.5 历史时间轴

- 横向滚动时间轴，X 轴为年份，Y 轴可分行显示不同国家
- 事件以时间条/节点形式展示，颜色区分国家
- 顶部：年份范围滑块缩放 + 国家筛选
- 冲突事件：节点显示红色警告图标，hover 展示冲突说明
- 右侧浮动按钮：「+ 新增事件」，弹出表单（年份、事件名、描述、关联人物多选、所属国家）

---

### 5.6 关系网络图

- 全屏力导向图（使用 D3.js 或 react-force-graph）
- 节点：人物头像圆形节点，悬停显示姓名
- 边：关系类型（盟友/对手/师徒/家族等），不同颜色线型
- 顶部控制栏：按国家过滤（多选）、关系类型过滤、缩放重置
- 点击节点：高亮该人物的所有关系，侧边显示人物卡片摘要

## 六、数据模型（参考）

```typescript
// 世界元数据
interface WorldMeta {
  name: string;
  description: string;
  createdAt: string;
  updatedAt: string;
}

// 国家/势力
interface Faction {
  id: string;
  name: string;
  color: string;       // 主题色（hex）
  description: string;
  era?: string[];      // 存在的历史时期
}

// 人物
interface Character {
  id: string;
  name: string;
  factionId: string;
  avatar?: string;     // base64 或 URL
  skills: string[];    // 技能标签
  traits: string[];    // 特质标签
  biography: string;
  birthYear?: number;
  deathYear?: number;
  relations: Relation[];
}

interface Relation {
  targetId: string;
  type: string;        // 'ally' | 'rival' | 'mentor' | 'family' | custom
  description?: string;
}

// 历史事件
interface HistoryEvent {
  id: string;
  year: number;
  title: string;
  description: string;
  factionIds: string[];
  characterIds: string[];
  location?: string;   // 地图图钉 id
}

// 地图图钉
interface MapPin {
  id: string;
  x: number;           // 相对坐标百分比
  y: number;
  label: string;
  type: 'country' | 'city' | 'landmark' | 'battlefield';
  factionId?: string;
  era?: string[];      // 哪些时期显示
  linkedCharacterIds?: string[];
  linkedEventIds?: string[];
}

// 完整导出结构
interface WorldData {
  meta: WorldMeta;
  factions: Faction[];
  characters: Character[];
  events: HistoryEvent[];
  pins: MapPin[];
  mapImage?: string;   // base64
}
```

| 项目 | 要求 |

| # | 问题 | 影响范围 | 优先级 |
|---|------|----------|--------|
| OQ-1 | 地图背景图片是否需要支持多张（不同世界地图）？还是单一世界只有一张主地图？ | 地图模块数据模型 | 高 |

| OQ-6 | 关系网络图的编辑入口：在人物卡片编辑页内填写关系，还是在关系图上直接拖拽连线？ | 关系图 UI 交互 | 低 |

## 九、成功指标

- 孩子能在 30 分钟内独立完成：上传地图 → 创建 3 个人物 → 记录 5 条事件 → 查看时间轴

- 冲突检测命中率：已知矛盾事件 100% 触发警告
- 父亲满意度：数据可随时以 JSON 形式完整备份
