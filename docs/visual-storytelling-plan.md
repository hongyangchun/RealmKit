# RealmKit 「世界圣典」视觉叙事方案

> **目标**: 让12岁的创造者感受到——我的世界是有生命的、在呼吸的、属于我的故事。
> 从「工具感」走向「叙事感」，从「数据展示」走向「情感体验」。

---

## 一、现状诊断

### 当前视觉资产清单

| 模块 | 现有视觉元素 | 叙事评分 | 关键缺失 |
|------|------------|---------|---------|
| **仪表盘** | Hero 横幅 + 迷你地图 + 势力卡片 | ★★☆ | 空状态有基础引导但缺乏故事沉浸感 |
| **世界地图** | GridCanvas 像素地图 + 图层系统 + 城市标记 | ★★☆ | 缺少事件叙事层，地图是"死的" |
| **人物系统** | 三国杀风格卡牌 + 力导向网络图 + 生命周期条 | ★★★ | 有叙事基因但弧光可视化不够深入 |
| **时间轴** | Canvas 绘制时间轴 + 筛选栏 | ★★☆ | 纯数据展示，缺乏时代感和叙事流 |
| **编年史** | 史书卡片列表 + AI 阅读器 | ★★★ | 最有叙事潜力的模块，视觉表现可再提升 |
| **全局** | LXGW WenKai 楷体 + 深蓝主题 + 羊皮纸底色 | ★★★ | 品牌基调已确立，需系统性深化 |

### 核心问题

1. **模块间叙事断裂** — 各页面独立运作，缺少贯穿始终的故事线索
2. **静态展示为主** — 数据可视化了，但没有"活"的感觉
3. **情感化空缺** — 空状态和过渡缺乏叙事引导
4. **视觉深度不足** — 只有一层信息展示，缺少渐进披露的叙事层次

## 三、五大策略方向详解

### 策略 1: 情感化空状态 — "每个空白都是故事的开端"

**现状**: 空世界引导卡片已存在（DashboardPage 的 `isWorldEmpty` 分支），但视觉表现偏"功能按钮"而非"故事邀请"。

**改进方案**:

#### 1.1 沉浸式首屏体验
- 将空状态从"卡片+按钮"升级为全屏叙事场景
- 背景采用缓慢漂浮的星尘粒子效果（CSS animation，不用 WebGL）
- 文案从"欢迎来到你的世界"升级为叙事引导：
  > "很久以前，这里什么都没有……直到你到来。"
- CTA 按钮改为"点亮第一颗星"而非"立即创建我的世界"

#### 1.2 渐进式世界构建叙事
- 世界种子向导（WorldSeedWizard）增加叙事包装
- 每一步选择用"故事语言"表达：
  - 选择势力数量 → "这片大陆上将崛起几个文明？"
  - 选择世界风格 → "你的故事发生在什么时代？"
  - 选择地形 → "大地的面貌由你决定"
- 生成完成后，展示一段过渡动画：从黑暗中逐渐亮起地图

#### 1.3 各模块专属空状态
- 地图空状态："这片土地等待着你的描绘" + 手绘笔触动画
- 人物空状态："第一个人即将踏上这片土地" + 剪影轮廓
- 时间轴空状态："时间尚未开始流动……" + 沙漏动画
- 编年史空状态："史册的扉页还是空白的" + 翻开的书页

```tsx
// 使用 MUI Fade + CSS keyframes 动画
// 背景粒子: 纯 CSS (多个小 div + animation-delay)
```

### 策略 2: 故事驱动的地图 — "地图是会讲故事的"

**现状**: GridCanvas 地图有地形+领地+城市+图钉图层，但纯空间展示，缺少时间维度和叙事维度。

**改进方案**:

#### 2.1 事件热力图层
- 新增 `event` 图层的可视化增强
- 在地图上叠加事件密度热力效果（半透明渐变光晕）
- 事件越多越密集的区域颜色越亮，呈现"历史热度"
- 点击热力区域弹出该位置相关的历史事件列表
- 实现：在 MapCanvas 上叠加一层 Canvas，用 radialGradient 绘制

#### 2.2 势力兴衰动画
- 新增地图"历史回放"模式
- 通过时间滑块控制，展示不同年份的势力版图变化
- 领地扩张/收缩用渐变过渡动画表现
- 配合事件标记，讲述"这个时代发生了什么"
- 数据来源：已有 `events` + `factions` 的年份信息

#### 2.3 地图标记叙事气泡
- MapPin 从简单图钉升级为叙事气泡
- 悬停显示：关联事件数量、最近的重大事件摘要
- 势力标记（城市）增加"统治时长"视觉指示器
- 城市类型用差异化视觉符号：
  - 国都：金色五角星 + 光晕
  - 要塞：盾牌图标
  - 港口：船锚图标
  - 圣地：神圣光环效果

#### 2.4 地图氛围层
- 不同地形类型增加环境氛围效果
- 海洋区域：微弱的波光效果（CSS animation）
- 森林区域：轻微的绿色光晕
- 雪峰：白色闪烁效果
- 让地图"活"起来，而非静态像素格

**技术实现**:

```tsx
// 增强: src/components/map/MapPin.tsx → NarrativeMapPin.tsx
```

### 策略 3: 人物视觉深度 — "每个角色都是一个故事"

**现状**: 人物卡牌已有三国杀风格、势力色带、生命周期条。关系网络图有力导向布局。

**改进方案**:

#### 3.1 角色弧光可视化
- 在 CharacterDetailPanel 中增加"角色弧光"时间线
- 横轴：角色生命周期（出生 → 死亡/至今）
- 纵轴：角色状态/权力/影响力（根据参与事件的性质推算）
- 关键事件在弧光上标注为转折点
- 弧光颜色跟随所属势力

#### 3.2 关系网络叙事增强
- RelationGraph 连线增加视觉语义：
  - 盟友关系：绿色实线 + 微弱光晕
  - 敌对关系：红色锯齿线
  - 血缘关系：金色双线
  - 师徒关系：紫色虚线 + 箭头
- 节点悬停时高亮其所有关系连线，其他淡出
- 关系强度用线条粗细表达（关联事件数量 → 线条越粗）
- 增加"关系故事"气泡：悬停连线显示两人之间发生了什么

#### 3.3 人物卡牌视觉升级
- 卡牌边框增加势力纹样装饰（SVG pattern）
- 生命周期条增加事件标记点（小菱形），悬停显示事件名
- 技能区域增加类型色彩区分：
  - 主动技能：暖色
  - 被动技能：冷色
  - 特殊技能：金色
- 冲突徽章（ConflictBadge）增加脉冲动画效果，暗示"这里有故事"

#### 3.4 角色详情叙事面板
- CharacterDetailPanel 增加叙事模式切换
- 数据模式（当前）：表格化信息展示
- 叙事模式：以人物传记形式展示，像读史书一样
- AI 生成的角色小传可以在此展示

### 策略 4: 时间轴叙事流 — "时间是故事的主角"

**现状**: TimelineCanvas 用 Canvas 绘制水平时间轴，有事件标记和筛选功能。

**改进方案**:

#### 4.1 时代色彩映射
- 根据事件密度和势力变化，将时间轴划分为"时代"
- 每个时代用不同的背景色带：
  - 开创期（事件少）：淡蓝，象征黎明
  - 繁荣期（事件多，势力稳定）：金色，象征盛世
  - 战乱期（多势力冲突）：暗红，象征战火
  - 衰落期（势力减少）：灰色，象征落幕
- 时代名称由 AI 或用户自定义

#### 4.2 时间轴事件叙事标记
- 事件标记从简单圆点升级为差异化图标：
  - 战争：交叉剑
  - 联盟：握手
  - 建城：城堡
  - 逝世：花环
  - 加冕：王冠
- 重大事件用更大的标记 + 金色边框突出
- 连续相关事件用虚线连接，展示因果关系

#### 4.3 编年史视觉层深度
- ChronicleReader 增加"古书翻页"效果
- 文字逐行淡入，模拟书写过程
- 章节间增加装饰性分隔符（古典花纹 SVG）
- 编年史卡片增加"书脊"效果（左侧色带 + 纹理）

### 策略 5: 微交互叙事 — "每个交互都在讲故事"

**改进方案**:

#### 5.1 统一过渡动画语言
- 页面切换：淡入淡出 + 轻微缩放（`transform: scale(0.98) → 1`）
- 卡片悬停：微弱上浮 + 光影变化（已有，保持一致）
- 弹窗打开：从触发元素位置展开
- 列表项进入：逐项延迟淡入（stagger animation）

#### 5.2 情感反馈动效
- 创建新势力：地图上该势力领地"亮起"的扩散效果
- 创建新人物：人物头像从模糊到清晰
- 删除确认：轻微震动反馈（mobile）

## 六、设计原则

1. **叙事先于功能** — 每个视觉决策先问"这讲述了什么故事？"
2. **渐进披露** — 宏观看全局，中观看关系，微观看细节
3. **一致的世界观** — 所有模块共享同一套视觉语言和叙事基调
4. **年龄适配** — 12岁的受众需要直观的视觉提示，避免过于抽象
5. **性能优先** — 动画使用 CSS transforms/opacity，避免触发重排
6. **可访问性** — 所有色彩对比度满足 WCAG AA，动画支持 `prefers-reduced-motion`
