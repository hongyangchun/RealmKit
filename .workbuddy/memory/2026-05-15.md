## 编辑表单下拉菜单无法更改 Bug 修复

根因：CharacterPage 表单弹窗使用自定义 `zIndex: 9999` 的 `<Box>` 实现，MUI `<Select>` 的 `<Menu>` 通过 Portal 渲染到 body，其默认 z-index 远低于 9999，导致下拉菜单被弹窗遮罩层遮挡，无法选中选项。

修复：将自定义 Box 弹窗替换为 MUI `<Modal>` 组件。Modal 自动管理 z-index 层级（使用 MUI 主题的 z-index 系统），Select 的下拉菜单能正确显示在 Modal 之上。同时 handleEdit 现在会先关闭 Drawer 再打开表单。

## 时间轴缩放不生效 Bug 修复

根因：TimelineCanvas.tsx 中 `scale` 状态在按钮点击时正确更新（百分比数字变化），但没有任何 CSS transform 使用该值——视觉上完全无变化。修复：在外层滚动容器内部新增包裹 Box，应用 `transform: scale(${scale})` + `transformOrigin: 'top left'`，同时通过 `width/height: ${100/scale}%` 补偿缩放后容器的实际占位大小。TypeScript 检查零错误。

## 世界种子生成器势力归属 Bug 修复

根因：`worldSeedGenerator.ts` 的 `generate()` 方法用 `faction_${i}` 作为占位 ID（如 `'faction_0'`），人物/事件/关系的引用都使用这些占位值。但 `worldStore.ts` 的 `applyWorldSeed()` 用势力**名称**建立映射表（`factionNameToId[f.name]`），导致查找时 `factionNameToId['faction_0']` 返回 undefined，人物 `factionId` 保留无效占位字符串，与真实势力 ID 无法匹配。

修复方案：改用占位 ID 格式建立映射（与生成端一致）：
- `worldStore.ts` — `factionNameToId` → `factionPlaceholderToId`，key 改为 `faction_${i}`；`charNameToId` → `charPlaceholderToId`，key 改为 `char_${idx}`
- `WorldSeedPreview.tsx` — 预览面板的人物势力颜色查找也改为基于索引匹配

影响范围：人物归属势力、关系人物引用、事件势力/人物引用全部修复。TypeScript 零错误，Vite 构建成功。
