## 地形层"小正方形"问题修复（commit f51f6fd）
- **问题**：新建世界后，地形层只显示在一个小正方形区域，没有铺满整个地图画布
- **根因 1（mapGrid 尺寸残留）**：`applyWorldSeed` 中 `const mapGrid = s.data.mapGrid ?? DEFAULT_GRID` 会复用旧世界的 mapGrid 尺寸（如 84×60），但种子生成器的地形数据是 100×100。渲染时 grid.width/height 为旧值，导致地形数据超出网格范围不可见
  - 修复：强制使用 `{ width: 100, height: 100, cellSize: 10 }`，不再 `??` 回退
- **根因 2（地形层 opacity 太低）**：DEFAULT_LAYERS 中地形层 opacity=0.2，在羊皮纸底色 `#faf3e0` 上几乎看不出来
  - 草地 #27ae60 在 0.2 opacity + #faf3e0 底色 → 混合后 #d0e5c6，与纯底色差值仅 82/765
  - 修复：将地形层默认 opacity 从 0.2 提高到 1.0（DEFAULT_LAYERS 和 applyWorldSeed 重置处两处同步修改）
- **验证**：agent-browser 截图确认地形铺满整个画布，颜色鲜明
- 已推送到 GitHub（Cloudflare Pages 自动部署）

## D1 云同步排查与错误反馈增强

### 问题现象
- 用户在办公室多次创建新世界，回家后家里浏览器仍显示儿子前天的旧世界
- 两边都用同一个 Google 账号通过 CF Access 登录

### 排查结论
- **根因**：`fetch('/api/world')` 返回 500 + Cloudflare HTML 错误页面，说明 D1 数据库未正确配置（可能表未创建或 binding 未关联）
- 所有 POST/GET 请求均失败，但错误被 `catch` 静默吞掉，用户无任何感知
- 两边浏览器各自用 localStorage，互不影响，所以儿子创建的世界 A 在家里一直没被覆盖

### 代码修改
- `syncService.ts`：新增 `_lastError` 和 `_consecutiveFailures` 字段，`getLastError()`、`hasSyncProblem()` 方法
- `syncService.ts`：debounce 成功时清除错误，失败时记录错误信息和连续失败次数
- `syncService.ts`：`postWorld/postChronicles` 对 500 错误给出更明确提示（"D1 数据库可能未配置"）
- `syncService.ts`：`fetchWorld/fetchChronicles` 对 500 错误特殊处理，不再抛异常导致静默
- `SettingsDialog.tsx`：新增 `manualSyncStatus` state，「立即同步」按钮显示同步中/成功/失败状态
- `SettingsDialog.tsx`：当 `hasSyncProblem()` 为 true 时，在云同步区域显示红色错误 Alert，提示用户检查 D1 绑定

### 修复过程
- 用户确认 D1 未绑定到 Pages 项目 → 在 Dashboard 手动绑定
- 绑定后仍 500 → 发现 GitHub 自动部署不读取 wrangler.toml 的 D1 binding
- `wrangler.toml` 缺少 `pages_build_output_dir = "dist"` 字段，导致 wrangler CLI 忽略配置
- 补上该字段 + `npx wrangler pages deploy` 直接部署 → API 返回 200
- **关键经验**：Cloudflare Pages + D1 必须在 wrangler.toml 中声明 `pages_build_output_dir`，否则 D1 binding 不会注入到 Functions

### 数据保护
- D1 已清空（办公室调试数据不重要），回家后家里浏览器 initialSync 会自动将儿子的世界数据推送为 D1 初始源
