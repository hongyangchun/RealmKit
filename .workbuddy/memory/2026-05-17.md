# 2026-05-17

## 全项目设计分析报告

### 范围
对 ZZWorld Chronicle 全项目 79 个源文件进行综合设计分析，覆盖功能、交互逻辑、UI 外观样式、代码架构四个维度。

### 关键发现
- **功能完整度高**：角色→势力→城市→地图→时间轴→编年史闭环，世界种子向导降低冷启动成本
- **视觉风格统一**：古典「羊皮纸」风格（深蓝+米黄+金色），氛围感强
- **主要优化方向**：
  - 体验闭环：地图↔实体双向联动不足，undo/redo 无 UI 入口
  - 视觉精细：Hero 区域对比度不足，全局 Paper 圆角覆盖过粗
  - 工程基建：缺少 Error Boundary、Loading 态、React.memo 性能优化
  - 代码质量：大型组件需拆分、代码重复需提取

### 输出
- 完整分析报告写入 `docs/design-analysis.md`
- 按优先级排序 16 项优化建议（P0-P3）

## 地图居中修复 + 按钮布局合并

### 问题
1. 地图默认显示偏向屏幕左边，没有居中
2. 编辑地图按钮和缩放控制按钮分开放置，占用空间且不统一

### 根因分析
- GridCanvas 渲染时用 `baseScale = Math.min(canvasW/logicalW, canvasH/logicalH)` 缩放网格
- 宽屏下高度适配后，水平方向有空白，但网格从 `(0,0)` 开始绘制，没有居中偏移
- `resetView` 设 pan 为 `{0,0}`，也不是居中位置

### 修复
- GridCanvas 新增 `offsetX/offsetY` 计算：`(canvasW - renderedW) / 2`
- 首次渲染时设置 `panRef = {x: offsetX, y: offsetY}`，网格自动居中
- `defaultCenterRef` 存储居中位置，`resetView` 恢复到此位置
- 新增 `hasInitializedRef` 防止重复初始化
- 暴露 `hitTest(clientX, clientY)` 方法给父组件精确坐标映射
- MapViewer 的 `handleCanvasClick` 改用 `hitTest` 替代自行计算坐标

### 按钮合并
- 缩放（放大/缩小/还原）和「编辑地图」按钮合并为一个竖排工具栏
- 编辑地图按钮用分隔线和缩放按钮分开，用金色（#ffd54f）区分功能
- 去掉文字标签，改为 Tooltip 提示

## 新建世界地图联动修复（关键 Bug）

### Bug 1：initGrid 清空数据
- **根因**：`GridCanvas` 的 `useEffect` 在容器尺寸变化时，会将容器像素尺寸 ÷ cellSize 得到网格尺寸，如果与 mapGrid 的 100×100 不匹配就调用 `initGrid()` 创建空网格，**直接清空了 worldSeed 生成的地形和领地数据**
- **修复**：`initGrid` 逻辑改为仅当 mapGrid 不存在或无任何 cells 数据时才重新初始化

### Bug 2：Canvas 渲染时序问题（核心根因）
- **根因**：GridCanvas 内部有独立的 `ResizeObserver` 直接修改 `canvas.width/height` DOM 属性，这**不触发 React 重渲染**。渲染 useEffect 依赖的 props `width/height` 与实际 canvas DOM 尺寸不同步，导致首次渲染时 canvas 内容为空
- **修复**：移除 GridCanvas 内部的 `ResizeObserver`，改为在渲染 useEffect 中同步设置 `canvas.width = width; canvas.height = height`
- **Playwright 验证**：移除后初始渲染即显示地形（center: `rgba(82,190,128,255)` 绿色地形），不再需要 resize 才触发

### 其他修复
- 渲染逻辑：计算 `effectiveCellSize = cellSize × scale`，让 100×100 网格适配任意容器尺寸
- `getCellFromMouse`：鼠标坐标转换也使用相同的 effectiveCellSize
- **worldStore.ts `redo()`**：cell key 格式从 `${cell.x},${cell.y}` 改为 `${cell.layerId}:${cell.x},${cell.y}`
- **DashboardPage.tsx**：`handleGenerate` 完成后自动 `navigate('/map')` 跳转到世界地图页面

## 地图平移和缩放功能

### 改动文件
- **`src/types/index.ts`**：`DrawingTool` 类型从 `'brush' | 'eraser'` 扩展为 `'brush' | 'eraser' | 'pan'`
- **`src/components/map/GridCanvas.tsx`**：完全重写渲染和交互逻辑
- **`src/components/map/DrawingToolbar.tsx`**：新增 `PanToolIcon`「移动」工具按钮

### GridCanvas 核心变更
- **平移**：使用 `panRef`（`{x, y}`）存储偏移量，渲染时 `ctx.translate(pan.x, pan.y)`
- **缩放**：使用 `zoomRef`（`0.5~8`）存储缩放比例，渲染时 `ctx.scale(zoom, zoom)`
- **交互方式**：
  - 左键拖拽平移（当工具为 `pan` 或按住空格时）
  - 中键拖拽平移（任何工具下可用）
  - 滚轮缩放（以鼠标位置为中心，使用 `mouseX - (mouseX - pan.x) * (newZoom / oldZoom)` 公式）
- **坐标映射**：`getCellFromMouse` 反向应用变换 `(cssX - pan.x) / zoom` 后再除以 `effectiveCellSize`
- pan/zoom 存储在 `useRef` 中避免每帧触发 React 重渲染，手动调用 `renderCanvas()` 重绘
- **Space 键监听**：`window` 级别的 `keydown/keyup` 事件监听 `Space` 键，临时启用平移模式

## 地形类型视觉差异增强

### 问题
用户反馈 4 种地形类型（大陆、群岛、沙漠、冰原）生成的地图看起来差不多。

### 改动文件
- **`src/services/mapGenerator.ts`**：参数表 + 噪声配置 + 分类逻辑

### 变更详情
1. **TerrainParams 新增字段**：`noiseFrequency`、`noiseOctaves`、`maskEdge`
   - 群岛使用更高频率(0.045)和更多octave(5)，产生破碎小岛
   - 沙漠使用缓边缘(0.7)，陆地延伸更远
   - 大陆保持默认(0.02, 4, 0.9)

2. **参数大幅拉开差距**：
   - 群岛 waterLevel 从 0.48→0.55（更多水）
   - 沙漠 desertBias 从 0.6→0.85（更多沙漠）
   - 冰原 snowBias 从 0.5→0.7（更多雪），snowLevel 从 0.82→0.72

3. **classifyTerrain 增强**：
   - 冰原：浅水区有概率变冰面(snowPeak)，丘陵有概率被雪覆盖
   - 沙漠：沙滩直接变沙漠，山地有概率变沙丘，平地高湿度区保留绿洲
   - 各地形类型的特征更鲜明

## 图层显示/隐藏功能 Bug 修复

### 用户报告
- 城市层隐藏后城市没有隐藏
- 领土层隐藏后领土的名字还在

### 根因分析
排查全部 5 个图层的渲染方式后，发现两类问题：

1. **领土标签不受图层可见性控制**：GridCanvas.tsx 渲染势力名称标签时（第 307 行），只检查了 `factions.length > 0` 和缩放级别，没有检查 territory 图层的 `visible` 属性

3. **事件标记可见性检查风格不一致**：使用 `!== false` 而其他地方使用 `!layer?.visible`

- **GridCanvas.tsx**：领土标签渲染条件增加 `territoryLayer?.visible` 检查；事件标记统一为 `eventLayer?.visible`（去掉 `!== false`）

## 撤销/重做功能严重 Bug 修复

### 问题
世界地图的撤销(Ctrl+Z)和重做(Ctrl+Shift+Z)按钮功能异常，操作无法正确回退/重做。

### 根因分析（3 个 Bug）

#### Bug 1：undo() 不删除当前操作新增的格子
- **场景**：在空白格子上涂色后撤销，格子仍然留在画布上
- **根因**：`undo()` 只恢复 `previousCells`，但没有删除 `entry.cells` 中记录的本次操作新增的格子
- 空白处涂色 → `previousCells: {}` → undo 遍历空对象，什么都不做

#### Bug 2：redo() 对 paint 操作未清理旧格子
- redo 没有先清理 previousCells 再写入 entry.cells

#### Bug 3：连续操作链断裂
- 涂色 → 擦除 → undo 擦除(恢复) → undo 涂色（格子没被删除）

### 修复
重写 undo() 和 redo() 为两步操作：
- undo: 1)删除entry.cells 2)对erase额外清理 3)恢复previousCells
- redo: 1)清理previousCells 2)写入entry.cells

### 改动文件
- `src/store/worldStore.ts`：`undo()` 和 `redo()` 方法重写

## 世界地图默认不居中修复

### 问题
世界地图页面打开时，地图偏向左上角而不是视觉居中。

### 根因
GridCanvas 的 `position: absolute; top: 0; left: 0` 覆盖了整个 MapViewer 容器，但容器上方有 52px 工具栏、左侧有 200px 图层面板。GridCanvas 拿到的 width/height 是全容器尺寸，居中计算基于全容器而非可见区域，导致视觉上偏左上。

### 修复
- **`src/components/map/MapViewer.tsx`**：
  - ResizeObserver 计算尺寸时减去工具栏高度(52px)和图层面板宽度(200px)
  - GridCanvas 和城市标记 overlay 移入 wrapper Box（`position: absolute; top: 52; left: 200`），与可见区域对齐
  - 这样 GridCanvas 的居中计算就是基于可见区域的真正居中

## 对话框下拉菜单被遮盖问题修复

### 问题
新建事件对话框中的下拉菜单（关联势力/关联人物/关联城市）被遮盖，无法正常选择选项。

### 根因
1. **TimelineCanvas 手写浮层 z-index:9999** 遮盖了 MUI Select Portal 下拉菜单（默认 z-index:1300）
2. **CharacterPage Modal 容器 overflow:auto** 导致下拉菜单被裁切
3. **全项目所有 Select 组件缺少 MenuProps** 配置，依赖 MUI 默认行为在高 z-index 容器中不够用

### 修复内容

#### 1. EventForm.tsx — 重构 + MenuProps
- 表单结构改用 `DialogContent` + `DialogActions` 包裹（之前只有裸 Box）
- 三个 `TextField select` 添加 `SelectProps={{ MenuProps: { sx: { zIndex: 10001 }, disablePortal: false } }}`

#### 2. TimelineCanvas.tsx — 手写浮层改 MUI Dialog
- 移除 `position: fixed + zIndex: 9999` 的手写浮层
- 改为 MUI `<Dialog>` 组件，利用 MUI 内置 z-index 管理
- 新增 `DialogTitle` + 关闭按钮

#### 3. CharacterPage.tsx — Modal 改 Dialog + MenuProps
- `<Modal>` + `overflow: auto` 容器改为 `<Dialog>` + `<DialogTitle>`
- 移除不再使用的 `Modal` import
- CharacterForm.tsx 中势力选择和技能类型选择添加 MenuProps

#### 4. 全项目 Select 统一添加 MenuProps
- RelationForm.tsx（3 个 Select）
- CityForm.tsx（2 个 Select）
- ChronicleForm.tsx（1 个 Select）
- PinForm.tsx（1 个 Select）
- TimelineFilterBar.tsx（2 个 Select）
- CharacterPage.tsx 工具栏（1 个 Select）

- vite build：成功
