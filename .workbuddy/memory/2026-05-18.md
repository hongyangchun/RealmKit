## 音频系统运行时 Bug 修复

用户反馈点击页面时出现 `InvalidStateError`、`Cannot read properties of null`、`AudioContext is closed` 三类错误。

### 根因分析
1. **竞态条件**：`useAudio` 路由 useEffect 在 `activate()` 完成前调用 `setMusicState()`/`setAmbienceScene()` → MusicSystem/AmbienceSystem 尚未初始化 → null 操作崩溃
2. **ADSR 时间非单调**：ProceduralMusic 的 4 个渲染器中，音符 duration 很短时 `setValueAtTime` 的时间点可能早于 `linearRampToValueAtTime` 的终点 → `InvalidStateError`
3. **变量名 Bug**：MusicSystem.ts `createPlayingTrack()` 第 286 行引用裸变量 `gain` 而非 `gainNode`

### 修复措施
- **AudioManager.ts**：新增 `audioSystemReady` 标志 + `pendingMusicState`/`pendingAmbienceScene` 缓存机制，activate() 完成后回放缓存状态
- **ProceduralMusic.ts**：`renderBass`/`renderMid`/`renderTreble`/`renderHigh` 全部 4 个渲染器统一使用 `Math.min`/`Math.max` 保护 ADSR 时间点单调递增 + duration ≤ 0.01 跳过
- **MusicSystem.ts**：`gain` → `gainNode` 变量名修复
- **ProceduralAmbience.ts**：审查确认所有 ADSR 使用纯 linearRamp 序列，天然单调递增，无需修改

- `npx tsc --noEmit`：0 错误
- `npx vite build`：构建成功
