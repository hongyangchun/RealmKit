## 数据丢失 Bug 修复（commit 2421f21）
- **根因 1（编年史）**：`syncService.ts` 的 `initialSync()` 中编年史合并逻辑将 ISO 时间戳字符串与 localStorage 中完整 JSON 内容做字符串比较（`cloudChroniclesUpdatedAt > localChroniclesStr`），导致云端编年史几乎总是覆盖本地编年史
  - 修复：改为基于条目 ID 的合并策略——本地为主，补充云端中本地没有的条目
- **根因 2（竞态）**：`SyncLoader.tsx` 在 `await initialSync()` 返回后直接覆盖 localStorage，未检查本地数据在等待期间是否被修改
  - 修复：同步前记录 `localSnapshotUpdatedAt`，返回后二次读取 localStorage 最新 updatedAt，若本地被修改则跳过覆盖并推送本地数据到云端

## 地图铺满 Bug 修复 — ReadOnlyMapPreview initGrid 竞态覆盖
- **问题**：自动生成的地图（100×100）在仪表盘页预览时被 ReadOnlyMapPreview 的 GridCanvas 覆盖为容器尺寸（如 84×60），导致地图无法铺满容器
- **根因**：`GridCanvas.tsx` 的 initGrid useEffect 在 `mapGrid.cells` 为空或维度不匹配时调用 `initGrid()`。ReadOnlyMapPreview 在仪表盘页挂载时，世界尚未生成（cells 为空），其 GridCanvas 以容器尺寸（如 840/10=84, 604/10=60）调用 `initGrid`，覆盖了种子生成器设定的 100×100 网格
- **修复**：在 GridCanvas 的 initGrid useEffect 中增加 `readOnly` 判断——当 `readOnly=true` 时跳过 `initGrid` 调用，因为只读预览不应修改全局网格数据
- Playwright 自动化测试验证：生成世界后 mapGrid 保持 `{width: 100, height: 100}`，地图页面和仪表盘预览均铺满容器无深蓝边距

## 地形层铺满修复 — islandMask 导致边缘深水（commit 0ab36ae）
- **问题**：新建世界时，地形（陆地）只出现在地图中央的正方形区域，周围全是深水（深海），没有铺满整个 100×100 地图
- **根因**：`mapGenerator.ts` 的 `islandMask` 函数根据单元格到中心的距离计算遮罩，`mask` = `1 - (dist * edgeSharpness)^2`。边缘单元格的 mask ≈ 0，导致 elevation 被压到远低于 waterLevel（如 0.19 < 0.35），整个外围变成深水
- **修复**：
  - `continent`/`desert`/`tundra` 的 `maskEdge` 设为 `0`（约定：0 表示不应用 islandMask）
  - `archipelago` 的 `maskEdge` 从 1.1 降到 0.65
  - 适当降低各地形类型的 waterLevel（continent 0.25, desert 0.18, tundra 0.22, archipelago 0.48）
  - `generateTerrain` 中增加条件判断：`maskEdge > 0` 时才应用 islandMask，否则直接用原始高度
- **验证**：Playwright 测试确认陆地覆盖率 87.87%，水域 12.13%，边缘水域仅 7.8%
- 已推送到 GitHub（Cloudflare Pages 自动部署）
